Power BI

Vendor ticket tracking dashboard

Executive-facing reporting · Data visualization · Time intelligence

Overview

Built an executive-facing Power BI dashboard giving leadership real-time visibility into vendor ticket performance. The dashboard answers a single critical question at a glance: is the support backlog growing or shrinking? It is reviewed monthly by leadership to evaluate vendor output, prioritize outstanding items, and ensure high-priority tickets are being resolved on schedule.

Technical features

  • ✓ KPI cards for tickets opened, closed, and outstanding — month-over-month change calculated using DAX time intelligence functions
  • ✓ Conditional formatting with directional arrow icons for instant visual status without manual interpretation
  • ✓ Dual line chart tracking open vs. closed volume over time to surface trends across reporting periods
  • ✓ Bar chart showing aging of open tickets by days-open buckets (0–30, 30–60, 60–90+ days) to flag stalled items
  • ✓ Date slicer connected to a dedicated date table for month-level filtering with correct time intelligence behavior

The aging chart and conditional formatting were specifically designed so leadership can identify vendor performance problems in seconds — without needing to query source data or interpret raw numbers.

Power Automate

Issue intake & triage automation

Workflow automation · Form design · Structured data capture

Overview

Designed and built a full end-to-end intake and routing workflow used daily by subject matter experts to report system defects during an active vendor-led implementation. Submissions are automatically processed, routed, and logged — feeding a structured weekly triage meeting attended by agency and vendor leadership to determine whether each issue becomes a formal vendor ticket.

Technical features

  • ✓ Microsoft Form capturing problem description, business and customer impact, affected account volume, issue category, severity level, and file attachments
  • ✓ Automated confirmation email sent to submitter on receipt — no manual acknowledgment needed
  • ✓ Category-based routing logic directs each submission to the correct shared mailbox based on issue type selected
  • ✓ Conditional escalation path: Critical-severity submissions trigger separate management alerts automatically
  • ✓ All submission data written to a structured Excel table with dropdown validation lists for assignment (name, manager) and generated email prompts via column concatenation
  • ✓ Screenshot attachment location extracted from submission and stored as a reference link in the Excel record

The workflow eliminated manual intake steps and ensures every submission is captured, routed correctly, and available for triage review — with all context (screenshots, severity, impact) immediately accessible to the people making decisions.
