Power BI

Vendor ticket tracking dashboard

Anomaly detection · Trend analysis · Executive-facing reporting

Overview

Built an executive-facing Power BI dashboard designed to surface anomalies and deviations in vendor ticket activity over time. The core analytical question: does the data reflect expected behavior, or is something not adding up? Month-over-month trend analysis, aging breakdowns, and conditional indicators make it immediately clear when volume patterns shift or items stall beyond expected thresholds. Reviewed monthly by leadership to identify outliers and hold activity to documented commitments.

Technical features

  • ✓ KPI cards for tickets opened, closed, and outstanding — month-over-month change calculated using DAX time intelligence functions
  • ✓ Conditional formatting with directional arrow icons for instant visual status without manual interpretation
  • ✓ Dual line chart tracking open vs. closed volume over time to surface trends across reporting periods
  • ✓ Bar chart showing aging of open tickets by days-open buckets (0–30, 30–60, 60–90+ days) to flag stalled items
  • ✓ Date slicer connected to a dedicated date table for month-level filtering with correct time intelligence behavior

The aging buckets and directional indicators were built specifically to flag when activity falls outside expected patterns — turning a raw ticket log into a tool for spotting what doesn't fit.

Power Automate

Issue intake & triage automation

Workflow automation · Form design · Structured data capture

Overview

Designed and built an end-to-end structured data capture and routing workflow for logging system defects during an active vendor-led implementation. Beyond automating intake, the system ensures every reported issue is categorized, severity-flagged, and preserved with full context — creating a reliable, auditable record of problems as they emerge. That data feeds a weekly triage meeting where patterns across submissions inform decisions about escalation and vendor accountability.

Technical features

  • ✓ Microsoft Form capturing problem description, business and customer impact, affected account volume, issue category, severity level, and file attachments
  • ✓ Automated confirmation email sent to submitter on receipt — no manual acknowledgment needed
  • ✓ Category-based routing logic directs each submission to the correct shared mailbox based on issue type selected
  • ✓ Conditional escalation path: Critical-severity submissions trigger separate management alerts automatically
  • ✓ All submission data written to a structured Excel table with dropdown validation lists for assignment (name, manager) and generated email prompts via column concatenation
  • ✓ Screenshot attachment location extracted from submission and stored as a reference link in the Excel record

Structured, consistent data capture across 600+ submissions means the record is searchable and comparable over time — making it possible to spot recurring issue types, volume spikes, or categories that disproportionately affect customers, which is where the more interesting analytical questions start.

Power Query & Excel

Automated data cleansing & reporting pipeline

ETL / data transformation · Process automation · Compliance reporting

Overview

Rebuilt a manual monthly compliance reporting process as an automated data pipeline. The original method — hand-keying values across multiple spreadsheet sections — introduced compounding error risk into data submitted directly to a government system for regulatory reporting. Automating the transformation and validation steps removed that risk at the source, ensuring the numbers entering the compliance record reflect the actual data rather than what someone transcribed.

Impact

4 hrs

Monthly time before

~10 min

Monthly time after

>95%

Time reduction

Technical features

  • ✓ Power Query pipeline ingests the raw third-party downloaded report and performs all cleansing and transformation steps automatically on refresh
  • ✓ Unit conversion logic identifies records from Canadian provinces and converts values from liters to gallons as part of the transform step
  • ✓ Pivot table groups, sums, and categorizes the cleaned output — replacing manual formula-based calculations across multiple sheets
  • ✓ Output validated against previously submitted data to confirm correctness before deployment
  • ✓ Job aide created with screenshots and exported Power Query M code from the advanced editor — providing a reference for maintenance and a recovery path if the source report structure changes

When data feeds a government compliance system, the integrity of that data matters beyond efficiency. The validation step — checking automated output against previously submitted records — was specifically designed to confirm the pipeline wasn't introducing new errors before it replaced the manual process entirely.
